share switched networks

电子交换

  1. 源站给目标站发资源预留请求
  2. 交换机做准入控制，创建源站和目标站之间的专用物理电路
  3. 传输数据
  4. 拆除请求，交换机释放电路和带宽资源

优点：

  • 性能可预测，不会卡顿和丢包
  • 交换简单和快速
    缺点：
  • 延迟
  • 交换机故障会导致所有依托它的电路都中断
  • 资源专用导致突发流量下效率极低

数据包分组交换

  1. 每个分组（包）都自带目的端地址
  2. 交换机根据分组的目的地址将分组转发到对应的端口
  3. 配备缓冲区，以缓解瞬时过载
  4. 存储 - 转发机制，分组在网络中一跳一跳传输，每个交换机必须先完整接收一个分组，再存储到缓冲区，最后转发，不能边收边发

优点：

  • 网络资源利用率高
  • 实现简单
  • 鲁棒性强
    缺点：
  • 性能不可预测
  • 需要进行缓冲区管理与拥塞控制

虚电路分组交换

共享资源，按需分配
多源 + 单链路 + 按需发送

多人共用一条路，谁有数据谁先走，没数据就不占资源

分配方式 电路交换（固定复用） 统计复用（分组交换）
资源分配 提前固定分配，全程独占 动态按需分配，共享资源
用户占用 即使用户没数据，也占着资源 用户没数据时，资源给别人用
核心依据 无统计特性，必须独占 利用 “用户并发概率” 的统计规律
  1. High resource utilization 高资源利用率
  2. Supports bursty traffic 支持突发流量
  3. Cost-effective 成本效益高

协议分层与服务模型

分层：

  • 每一层实现一种服务
  • 通过自己层内部的操作
  • 依赖下层提供的服务

why

  • 处理复杂系统
  • 模块化：方便维护更新，对其他层透明

标准协议体系结构

两种标准：OSI 和 TCP/IP（现在 widely used）
Others：。。。

osi

A layer model, and flow structure
分层模型 + 数据流结构
Each layer performs a subset of functions
每层只完成一部分功能
Each layer relies on next lower layer
每层依赖下一层提供更基础的功能
Each layer provides services to higher layer
每层为上一层提供服务
Changes in one layer should not affect others
修改一层不影响其他层

  1. 物理层：Transfers bits across link（数据线，接口，，）
  • 定义通信链路的物理性质
  • 设备间的物理接口
  1. 数据链路层：Transfers frames across link把比特组成帧
  2. 网络层：Transfers packets across network负责源主机 → 目标主机的传输，对分组进行路由选择
  3. 传输层：Transfers data across connection应用进程之间的通信
  4. 应用层：Manages sessions between applications直接为用户应用程序提供服务

tcp/ip

  1. 应用层：给应用程序服务（网页、邮件、文件）（FTP, SMTP, HTTP）
  2. 传输层：程序到程序的数据传输：（TCP、UDP）
  3. 网络层：数据包传输：（IP）
  4. 数据链路层：相邻设备之间数据传输：（以太网，PPP）
  5. 物理层：数据线，物理线路传比特

沙漏模型：

  • 网络层只有一个协议：IP
  • 只要支持 IP，任何网络都能互通
  • 应用和底层技术分开，应用在哪都能用
  • IP 上面、下面都能独立创新
  • 缺点：IP 很难改（比如 IPv4 升级 IPv6 非常难

分层封装：

  • 每一层都会给数据加头部（控制信息）
  • 传输层把应用数据切成段（segment）
  • 加上：
    • 目标端口
    • 序号
    • 差错校验码
  • 组合起来就是 传输层 PDU（协议数据单元）

封装：

  • 从上到下传递
  • 层间数据传递时，每层都会添加自己的头部
  • 应用数据→ 加传输头→ 加网络头→ 加链路头→ 变成比特流发送

优缺点：
✅ 优点：

  • 降低复杂度
  • 更灵活
    ❌ 缺点：
  • 开销更大
  • 有些信息跨层用更高效，但分层不让用

互联网性能

  1. Delay 时延 —— 数据包从源到目的花多久
    Transmission delay 传输时延:把数据包 “推” 进链路花的时间
  • 传输时延 = 数据包长度 ÷ 链路带宽（发送速率）
    Propagation delay 传播时延: 1 个比特在链路上 “跑” 完全程的时间
  • 传播时延 = 数据包长度 ÷ 链路传输速度

    → 这两个由链路本身决定

Queuing delay 排队时延

  • Arrival process :
    • Average rate λ(每秒多少包)
    • Peak rate ρ(最高每秒多少包)
  • W = 平均等待时间
  • L = 队列里的平均包数

-L = A x W

Processing delay 处理时延 ：交换机处理一个数据包要花的时间（几乎没有）

→ 这两个由流量、交换机内部决定

  1. Loss 丢包 —— 数据包有没有弄丢

  2. Throughput 吞吐量 —— 一秒能传多少数据

  • R：链路传输速率（带宽），单位 bits/sec
  • F：文件大小，单位 bits
  • T：传输总时间
  • T = F / R + （propagation delay）
  • Average throughput = min{R, R’} = R：整条路径的吞吐量由最慢的那条链路决定
